My experience with Angular

  • Only AngularJS (Angular 1)
  • Very steep learning curve, complicated
  • Performance issues (digest cycles)
  • Most of work done in templates
  • Hard to say how to do things properly on a first try

My experience with React

  • After Angular, I wanted to try something fresh
  • First choice for my startup
  • Simple concepts, great community & tooling, hard to master
  • Only view, requires understanding of lots of additional libraries
  • HTML generated by JS, mixed View, logic and sometimes CSS in the same place
  • Redux helps, but is very verbose
  • Can't be used in very simple projects (JSX)

Single File Components

  • Creating components as strings is not fun
  • SFC has *.vue extension
  • Requires babel and vue-loader to work
  • HTML, JS and CSS in one file
  • Vastly reduces mental overhead, less files to look at
  • One of the greatest ideas in the Vue

                <template>

Reactivity

How Vue works under the hood?

  • All data objects are converted into properties
  • Each property is watched over for changes
  • When change occur, dependencies (HTML, computed) are updated
  • Templates are translated into render functions
  • Utilizes virtual DOM for performance
